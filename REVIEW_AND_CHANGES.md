# Revision Audit — Show Orchestration V1 Planning Bundle

## Revision basis

This revision incorporates only recommendations from `deep-research-report.md` that are materially applicable to the current local Codex + Claude + Git worktree pipeline. The detailed accept/defer/reject record is in `docs/REPORT_ADOPTION.md`.

## Previously hardened items retained

- Required schema versioning and rejection of unknown YAML fields.
- Separated analog and digital target kinds.
- Exact virtual-path reverse/seam gold vectors.
- Absent/no-contribution separated from intentional black.
- Exact deterministic composition and transition formulas.
- Mandatory `[start, end)` semantics and cue-local time.
- Split music features from adaptive selection.
- Prohibition on skip/xfail bypasses, test weakening, and test-aware production code.
- Software acceptance separated from hardware verification.

## New changes from the research report

- Added five single-authority contract documents for frames, composition, time, music, and quality gates.
- Added immutable G1–G8 cross-Phase goldens with SHA-256 manifest.
- Added five locked deterministic WAV fixtures and hash manifest.
- Added recursive strictness and rejection of numeric-string coercion to Phase 11.
- Added optional `gap_after_pixels` with exact unmapped-coordinate semantics to Phase 12.
- Added decision-table and `SelectionDecision.reason_code` evidence to Phase 16.
- Added a standardized Phase audit JSON schema and a committed-baseline verifier that checks contracts, manifests, locked hashes, tracked WAV fixtures, repair budgets, and clean Git state.
- Added differential repair budgets: 2/2/2/2/3/3/1.
- Added four segmented manifests for human checkpoints without losing automated dual-agent execution.
- Added offline and separate 300-second real-time software acceptance.
- Added artifact hash requirements and fixed evidence lists.
- Added minimal and full YAML authoring templates.
- Expanded hardware acceptance to record gap calibration, protocol compatibility, and measured node skew.

## Deliberately not added

- GitHub Actions, CODEOWNERS, third-party Action SHA pinning, and artifact attestation: the current target is a local PowerShell/worktree pipeline.
- Machine-learning genre or emotion classification.
- Automatic millimetre calibration.
- Blanket immutability of all tests; legitimate Phase-specific tests remain allowed and audited.
- Software claims of teacher-approved aesthetics or hardware completion.

## Result

The package now places reusable semantics and evidence outside individual Phase discretion. The dual-agent pipeline can strongly verify software contracts and regressions, while final screen-to-wall appearance remains a recorded hardware/human acceptance task.
